# 小林畳商店 ウェブサイト 要件定義書

## 1. 概要

畳専門店「小林畳商店」を、Astro + Cloudflare Pages + Cloudflare Workersの構成でホストする。Eコマース機能は不要で、問い合わせ（見積依頼）への導線がサイトの主目的。

## ウェブサイトの基本方針

1. スマートフォン対応（レスポンシブデザイン）
2. 表示速度の改善（Core Web Vitals最適化）
3. SEO・構造化データの強化で地域検索での上位表示を図る
4. 問い合わせ促進（UX改善）
5. 管理性の継続的改善・維持（更新・保守のしやすさ）
6. 2言語対応（日本語・英語）
7. 職人の仕事が伝わるビジュアルとなるようブランディングを考慮

## 2. サイト構成

トップページに主要情報を集約し、詳細コンテンツは個別ページに分離。ユーザーが必要な情報に段階的にアクセスできる構成。

### 2.1 トップページ（/）

サイトの顔。「何をやっている店か」「なぜ選ぶべきか」がスクロールで完結する構成。

| セクション | 内容 | 備考 |
|------------|------|------|
| ヒーロー | キャッチコピー + 写真 + 行動喚起 | 職人の丁寧な仕事を伝えるメッセージ |
| 強み・特徴 | 一貫対応 / 中間マージンなし / 一級畳技能士 | ビジュアル付きで訴求 |
| サービス概要 | 畳工事・襖工事・内装工事の概要 | 詳細は /services へリンク |
| 施工例ハイライト | 3〜4件をピックアップ | 写真+キャプション付き、詳細は /works へ |
| 行動喚起 | 電話 + 問い合わせフォームへのリンク | LINE等は先方確認後に追加検討 |

### 2.2 サービス詳細（/services）

提供サービスの詳細を掲載。

| カテゴリ | 内容 |
|----------|------|
| 畳工事 | 新畳、表替え、裏返し、乾燥、縁なし畳（琉球畳） |
| 襖工事 | 新襖、襖張替え、障子張替え |
| 内装工事・リフォーム | クロス張替え、簡単な大工仕事（建具修繕・床修理） |

- 各サービスに対応する施工例写真を含む

### 2.3 畳の知識（/knowledge）

畳に関する教育コンテンツを掲載。店主のこだわりを伝える重要なページであり、UIで読みやすく整理。

| トピック | 内容 |
|----------|------|
| 畳の構造 | 畳床（藁床・藁サンド・建材床）、畳表、畳縁の解説 |
| 畳の種類 | 縁付き畳、縁なし畳、龍鬢表（床の間用） |
| 畳を選ぶポイント | 国産イグサ vs 中国産イグサの比較、用途別おすすめ |
| 畳替えのタイミング | 季節別・行事別の畳替え例、店主のメッセージ |

- アコーディオンまたはタブUIで長文の圧迫感を軽減
- 畳のサイズ目安表をテキスト/テーブル形式で掲載
- 内容の取捨選択は先方と要相談

### 2.4 施工例（/works）

施工事例を写真とキャプション付きで掲載。

- 写真 + キャプション（工事種別、施工内容の簡単な説明）
- カテゴリ分け（畳工事 / 襖工事 / 内装工事）
- Before/After形式が可能なら訴求力が向上（先方確認）

### 2.5 会社概要・アクセス（/about）

小林畳商店の基本情報を掲載するページ。

詳細な会社情報は [about-store.md](./about-store.md) を参照。

**このページに含める要素**:

- 会社基本情報（about-store.mdの内容）
- Google Maps埋め込み
- 信頼できる印象を与えるため代表者の写真・プロフィール（職人としての経歴等）を含む

### 2.6 お問い合わせ（/contact）

- 電話番号の表示（タップで発信可能に）
- 問い合わせフォーム（Cloudflare Workersで実装）
- フォーム項目（想定）：名前、電話番号、メールアドレス、お問い合わせ内容（自由記述）、希望するサービス（選択式）

## 3. 非機能要件

### 3.1 パフォーマンス

- Lighthouse Performance スコア 90以上を目標
- Astroのゼロ JS デフォルト出力を活かし、必要な箇所のみインタラクティブに
- 画像はWebP/AVIFで最適化し遅延読み込み対応

### 3.2 レスポンシブデザイン

- モバイル・PC同等の設計
- ブレイクポイント: モバイル（〜767px）/ タブレット（768〜1023px）/ デスクトップ（1024px〜）
  - 詳細は [technologies.md](./technologies.md) のブレイクポイント定義を参照
- タップターゲットは44px以上（電話番号、行動喚起ボタン等）

### 3.3 アクセシビリティ

- 適切なセマンティックHTML
- alt属性の設定（施工例写真含む）
- コントラスト比の確保
- キーボードナビゲーション対応

### 3.4 SEO

- LocalBusiness構造化データの実装（JSON-LD）
- ページごとのtitle / meta description
- 適切な見出し階層（h1〜h3）
- サイトマップ自動生成（Astro標準機能）
- OGP / Twitter Card メタタグ
- 旧URL → 新URLの301リダイレクト
- Google Search Console登録・サイトマップ送信
- Googleビジネスプロフィールの URL更新
